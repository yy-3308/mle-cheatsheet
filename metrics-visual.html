What each metric means

Metric
Plain English
Business sense (Expedia)
Accuracy
Out of everyone, how many did I get right?
Misleading when imbalanced. Always predicting "won't book" = 99% accuracy but useless.
Precision
When I said YES, how often was I right?
Fewer false alarms. Don't waste marketing budget on non-bookers.
Recall
Of all real YESes, how many did I catch?
Fewer missed real bookers. Missing a real booker = lost revenue. Usually more important at Expedia.
ROC-AUC
If I pick one real and one non-booker, how often does my model rank the booker higher?
Best overall measure. 0.5 = random, 1.0 = perfect. Use instead of accuracy for imbalanced data.

Regression. How much variance does the model explain?

Which metric to use

Predicting a category? → Classification
  → Classes balanced: Accuracy
  → Imbalanced: ROC-AUC
  → False alarms costly: Precision
  → Missing real cases costly: Recall
  → Need one balanced number: F1

Predicting a number? → Regression
  → Easy to explain: MAE
  → Big errors are especially bad: RMSE
  → Overall quality: R²

Precision vs Recall — the tradeoff

You can't maximize both at once. Catching more real bookers (↑ recall) means more false alarms (↓ precision).

→ Missing a real booker is worse → optimize Recall
→ Wasting budget on non-bookers is worse → optimize Precision

For Expedia → Recall is usually more important.

ROC-AUC — Area Under the Curve

The one-liner: If I pick one real booker and one non-booker at random, what's the chance my model scores the booker higher?

AUC = 0.5 → random guessing  |  AUC = 1.0 → perfect
● Perfect model — hugs top-left, AUC = 1.0
● Good model — bows toward top-left, AUC ≈ 0.85
● Random — diagonal line, AUC = 0.5

Rule of thumb: 0.7+ decent  |  0.8+ good  |  0.9+ very good
