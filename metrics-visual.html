What each metric means — in plain English & business terms

Metric
In plain English
Business sense (Expedia example)
Accuracy
Out of everyone, how many did I get right?
Misleading when data is imbalanced. 99% of users don't book → always saying "won't book" gives 99% accuracy but is useless.
Precision
When I said YES, how often was I actually right?
Important when false alarms are costly. Sending 1000 discount emails to non-bookers wastes marketing budget → you want high precision.
Recall
Out of all real YESes, how many did I catch?
Important when missing a real case is costly. Missing a real booker = lost revenue → you want high recall. Usually more important than precision for Expedia.
ROC-AUC
If I pick one real booker and one non-booker, how often does my model rank the booker higher?
Best overall measure of model quality. 0.5 = random guessing, 1.0 = perfect. Use this instead of accuracy when classes are imbalanced.

Other important metrics

Metric
Formula
When to use
F1 Score
2 × (Precision × Recall) / (Precision + Recall)
When you can't choose between precision and recall — gives one balanced number. Good for imbalanced classes.
MAE
Average of |actual − predicted|
Regression. Easy to explain — "on average, my revenue prediction is off by $200." Treats all errors equally.
RMSE
√(average of (actual − predicted)²)
Regression. Penalizes large errors more than MAE. Use when big mistakes are especially bad.
R²
1 − (model error / baseline error)
Regression. How much variance does the model explain? 1.0 = perfect, 0 = no better than predicting the mean, negative = worse than mean.

Which metric to use — quick guide

Predicting a category (book / not book)? → Classification
  → Classes balanced: Accuracy
  → Classes imbalanced: ROC-AUC
  → False alarms costly: Precision
  → Missing real cases costly: Recall
  → Need one balanced number: F1 Score

Predicting a number (revenue, clicks)? → Regression
  → Easy to explain to business: MAE
  → Big errors are especially bad: RMSE
  → Overall model quality: R²

Precision vs Recall — the tradeoff

You can't maximize both at the same time. Catching more real bookers (↑ recall) means more false alarms (↓ precision). The right balance depends on the business cost.

→ If missing a real booker is worse: optimize for Recall
→ If wasting budget on non-bookers is worse: optimize for Precision

For Expedia affiliate revenue prediction → Recall is usually more important.

ROC-AUC — Area Under the Curve

ROC = Receiver Operating Characteristic — the curve plotted at every threshold
AUC = Area Under the Curve — one number summarizing the whole curve

The one-liner: If I pick one real booker and one non-booker at random, what's the chance my model scores the booker higher?
X axis (FPR) — False Positive Rate: how many non-bookers did we wrongly flag?
Y axis (TPR) — True Positive Rate (= Recall): how many real bookers did we catch?

● Perfect model — hugs top-left corner, AUC = 1.0
● Good model — bows toward top-left, AUC ~0.85
● Random model — diagonal line, AUC = 0.5 (coin flip)

Rule of thumb: 0.7+ decent  |  0.8+ good  |  0.9+ very good (check for leakage!)
